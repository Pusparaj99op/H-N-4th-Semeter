Practical No. 05 – Setting up a Peer-to-Peer (P2P) Network

Aim

To establish a Peer-to-Peer (P2P) network between two computers, configure static IP addresses, enable file sharing, and verify resource sharing between the peers.

Theory

Peer-to-Peer (P2P) Network: A network architecture in which every workstation (peer) has equal capabilities and can act as both a client and a server. No dedicated central server is required.

Key Characteristics:

  • Each computer controls its own resources and user accounts.
  • Resources (files, printers, internet) can be shared directly between peers.
  • Best suited for small networks of 2–10 computers.
  • All computers must be in the same Workgroup (default: WORKGROUP).

Connection Methods for P2P:

  • Direct Connection: Using a Cross-Over UTP cable between two PCs' Ethernet ports.
  • Via Switch/Hub: Both PCs connected to a common switch using straight-through cables.
  • Wireless: Using the same Wi-Fi network or an ad-hoc wireless connection.

IP Configuration for P2P: Both PCs must have IP addresses on the same subnet and no default gateway is strictly required. Example: PC1 = 192.168.1.1/24 and PC2 = 192.168.1.2/24.

Apparatus / Requirements

  • 2 Computer systems running Windows OS
  • 1 Cross-Over UTP cable (for direct connection) OR 1 Switch + 2 Straight-Through cables
  • Network Interface Cards (NICs) in both computers
  • A test folder/file to transfer between the computers

Procedure

  1. Physical Setup:
    • Connect the two PCs using a crossover cable directly, OR
    • Connect both PCs to a network switch using straight-through cables.
  2. Set a Static IP on PC1:
    • Go to Control Panel → Network and Sharing Center → Change adapter settings.
    • Right-click the Ethernet adapter → Properties → Select IPv4 → Properties.
    • Set: IP = 192.168.1.1, Subnet Mask = 255.255.255.0. Leave Gateway blank. Click OK.
  3. Set a Static IP on PC2:
    • Repeat the same steps on PC2.
    • Set: IP = 192.168.1.2, Subnet Mask = 255.255.255.0. Click OK.
  4. Set Same Workgroup on Both PCs:
    • Right-click This PC → Properties → Change settings → Change.
    • Ensure both PCs have the same Workgroup name (e.g., WORKGROUP). Restart if changed.
  5. Test Connectivity:
    • On PC1, open CMD and run: ping 192.168.1.2
    • If 4 packets are received with 0% loss, the connection is successful.
  6. Enable Network Discovery & File Sharing:
    • On both PCs: Control Panel → Network and Sharing Center → Change advanced sharing settings.
    • Under the Private profile, turn on Network discovery and File and printer sharing.
    • Click Save changes.
  7. Share a Folder from PC1:
    • Create a folder (e.g., C:\P2P_Share). Place a test file inside it.
    • Right-click the folder → Properties → Sharing → Share…
    • Add Everyone, set permission to Read/Write → click Share.
    • Note the network path shown (e.g., \\PC1\P2P_Share).
  8. Access Shared Folder from PC2:
    • Open File Explorer on PC2. In the address bar type: \\192.168.1.1 and press Enter.
    • The shared folder from PC1 should be visible. Open it and copy/paste a test file.
  9. Verify the file is transferred successfully on both machines. Document the output.

Result

A Peer-to-Peer network was successfully established between two computers. Static IP addresses (192.168.1.1 and 192.168.1.2) were configured on both machines. Connectivity was verified using the ping command with 0% packet loss. File sharing was enabled and a folder was successfully shared and accessed from the remote peer, confirming bidirectional resource sharing over the P2P network.
