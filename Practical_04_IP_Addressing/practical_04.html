Practical No. 04 – IP Addressing and Subnetting

Aim

To understand IPv4 addressing, IP address classes, subnet masks, and to perform subnetting calculations. Also to manually configure static IP on a Windows system and verify connectivity.

Theory

IPv4 Address: A 32-bit logical address assigned to every device on an IP network, written in dotted-decimal notation (e.g., 192.168.1.1). Each of the four groups (octets) ranges from 0 to 255.

Private IP Address Ranges (RFC 1918):

  • Class A: 10.0.0.0 – 10.255.255.255
  • Class B: 172.16.0.0 – 172.31.255.255
  • Class C: 192.168.0.0 – 192.168.255.255

Subnet Mask: A 32-bit number used to divide an IP address into Network and Host portions. Bits set to 1 represent the network, bits set to 0 represent hosts. Example: /24 = 255.255.255.0 in binary: 11111111.11111111.11111111.00000000

Subnetting: The process of dividing a single network into smaller sub-networks to improve performance and security. Key formulas:

Number of Subnets = 2n  (n = number of borrowed bits)

Hosts per Subnet = 2h – 2  (h = remaining host bits; subtract 2 for network and broadcast addresses)

Subnetting Example: Network: 192.168.10.0/24 — Divide into 4 equal subnets.

  • Need 4 subnets → borrow 2 bits: 2² = 4
  • New CIDR: /26  |  New Subnet Mask: 255.255.255.192
  • Block size: 256 – 192 = 64  |  Hosts per subnet: 2⁶ – 2 = 62

Apparatus / Requirements

  • Computer System with Windows OS
  • Command Prompt (cmd.exe)
  • Scientific calculator / Subnet Calculator tool
  • Active LAN connection (for IP assignment and testing)
  • Cisco Packet Tracer (optional, for network simulation)

Procedure

  1. Subnetting Exercise (Manual Calculation):
    • Given network: 192.168.1.0/24. Task: divide into 8 subnets.
    • Borrow 3 bits (2³ = 8). New mask: /27 = 255.255.255.224.
    • Block size: 256 – 224 = 32. Hosts per subnet: 2⁵ – 2 = 30.
    • List all 8 subnets (0, 32, 64, 96, 128, 160, 192, 224) with their host ranges and broadcast addresses.
  2. Configure Static IP on Windows:
    • Open Control Panel → Network and Sharing Center → Change adapter settings.
    • Right-click on the Ethernet adapter → Properties.
    • Select Internet Protocol Version 4 (TCP/IPv4) → click Properties.
    • Select "Use the following IP address".
    • Enter: IP = 192.168.1.10; Subnet Mask = 255.255.255.0; Default Gateway = 192.168.1.1.
    • Enter Preferred DNS: 8.8.8.8 (Google DNS). Click OK.
  3. Open Command Prompt. Type ipconfig to verify the configured IP address.
  4. Ping the default gateway: ping 192.168.1.1 to verify local connectivity.
  5. Ping another device on the same subnet to confirm end-to-end reachability.
  6. Type ipconfig /all — note the IPv4 address, subnet mask, and gateway settings.
  7. After the experiment, restore the adapter to "Obtain an IP address automatically" (DHCP).
  8. Document all subnet tables and screenshot of ipconfig output.

Result

IPv4 address classes, subnet masks, and CIDR notation were studied. Subnetting calculations were successfully performed for 192.168.10.0/24 (into 4 subnets) and 192.168.1.0/24 (into 8 subnets). A static IP address was manually configured on a Windows system, and connectivity to the gateway was verified using the ping command.
