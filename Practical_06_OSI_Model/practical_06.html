Practical No. 06 – Study of the OSI Reference Model

Aim

To study the OSI (Open Systems Interconnection) reference model in detail, understand the function of each of its seven layers, and observe protocol-layer relationships using a packet analyser.

Theory

OSI Model: The OSI model is a conceptual framework developed by ISO (International Organization for Standardization) in 1984 that standardises the communication functions of a network system into seven distinct layers. It ensures interoperability between different hardware and software products from different vendors.

Detailed Description of Each Layer:

Layer 7 – Application: The topmost layer that provides network services directly to the end user's application (browser, email client). It handles protocols like HTTP for web browsing, SMTP for email, and FTP for file transfers.

Layer 6 – Presentation: Responsible for data translation, encryption, and compression. It ensures data from the application layer of one system can be read by the application layer of another. Handles format conversion (e.g., EBCDIC to ASCII) and encryption (SSL/TLS).

Layer 5 – Session: Manages establishing, maintaining, and terminating communication sessions between applications. Handles authentication, reconnection after interruption, and synchronisation (checkpoints).

Layer 4 – Transport: Provides reliable end-to-end data delivery between processes. TCP provides error checking, sequencing, and flow control (connection-oriented). UDP is faster but connectionless with no guaranteed delivery.

Layer 3 – Network: Handles logical addressing (IP addresses) and routing — determining the best path for data to travel from source to destination across multiple networks. Routers operate at this layer.

Layer 2 – Data Link: Responsible for physical addressing (MAC addresses), framing, and error detection (CRC). Split into two sub-layers: LLC (Logical Link Control) and MAC (Media Access Control). Switches operate here.

Layer 1 – Physical: Defines the physical medium for data transmission — electrical signals on copper, light pulses on fibre, radio waves for Wi-Fi. Defines voltages, pin layout, cable specifications, and bit rate.

Apparatus / Requirements

  • Computer System with Windows OS
  • Wireshark (free network packet analyser — www.wireshark.org)
  • Active network connection (LAN/Wi-Fi)
  • Cisco Packet Tracer (optional, for OSI simulation)

Procedure

  1. Draw the OSI model with all 7 layers and their data units in the journal.
  2. Install and open Wireshark on the computer.
  3. Select the active network interface (Ethernet or Wi-Fi) from the start screen.
  4. Click the blue shark-fin icon (or press Ctrl+E) to start packet capture.
  5. Open a browser and visit http://www.google.com to generate HTTP traffic.
  6. Return to Wireshark and click the red stop button after a few seconds.
  7. In the filter bar, type http or tcp and press Enter to filter relevant packets.
  8. Click on a captured packet. In the Packet Details pane, expand each protocol layer:
    • Frame → Physical layer info (frame size, arrival time)
    • Ethernet II → Data Link layer (source/destination MAC addresses)
    • Internet Protocol → Network layer (source/destination IP addresses, TTL)
    • Transmission Control Protocol → Transport layer (port numbers, sequence/ack numbers)
    • Hypertext Transfer Protocol → Application layer (HTTP GET request, headers)
  9. Record MAC addresses, IP addresses, port numbers, and HTTP request/response headers in your journal.
  10. Take a screenshot of the packet details panel and document the findings.

Result

The OSI Reference Model was studied in detail. All 7 layers — Physical, Data Link, Network, Transport, Session, Presentation, and Application — were understood along with their functions, protocols, data units, and associated network devices. Practical demonstration using Wireshark confirmed the layered protocol structure in a real HTTP packet, where Ethernet (L2), IP (L3), TCP (L4), and HTTP (L7) layers were clearly identified.
