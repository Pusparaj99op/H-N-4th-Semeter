Practical No. 09 – Study of Network Devices

Aim

To study and understand various hardware devices used in computer networks — namely Repeater, Hub, Bridge, Switch, Router, Gateway, and NIC — including their characteristics, OSI layer, and use cases.

Theory

Network devices are hardware components that connect computers and other devices together to share resources and communicate. Each device operates at a specific layer of the OSI model.

1. NIC – Network Interface Card

OSI Layer: Physical & Data Link (Layer 1 & 2). A hardware card installed in a computer that provides the physical connection to a network. Each NIC has a unique 48-bit MAC (Media Access Control) address burned in by the manufacturer. Available in wired (Ethernet) and wireless (Wi-Fi) forms. Most modern computers have the NIC integrated on the motherboard.

2. Repeater

OSI Layer: Physical (Layer 1). Receives a signal, amplifies/regenerates it, and retransmits it to extend the network over a greater distance. It has no intelligence — it repeats all signals including noise. Does not filter traffic. Commonly used to extend Ethernet segments beyond their maximum cable length (e.g., 100 m for CAT5e).

3. Hub

OSI Layer: Physical (Layer 1). A multi-port repeater. When a frame arrives on one port, it broadcasts it to all other ports regardless of destination. This creates a shared collision domain — only one device can transmit at a time. Available in active (powered) and passive (no power) types. Largely obsolete — replaced by switches. Operate in half-duplex mode only.

4. Bridge

OSI Layer: Data Link (Layer 2). Connects two LAN segments and uses a MAC address table to forward frames only to the correct segment, reducing unnecessary traffic. It divides a network into collision domains. Smarter than a hub but simpler than a switch. Most commonly two-port devices. Bridges are the predecessors of modern switches.

5. Switch (Layer 2 Switch)

OSI Layer: Data Link (Layer 2). A multi-port bridge. Maintains a MAC address table (CAM table) and forwards frames only to the specific destination port, creating separate collision domains for each port. This dramatically improves performance over hubs. Supports full-duplex communication. Most modern LANs are entirely switch-based. Layer 3 switches can also perform routing.

6. Router

OSI Layer: Network (Layer 3). Connects multiple networks (e.g., LAN to WAN/Internet). Uses a routing table and IP addresses to determine the best path to forward packets. Each interface on a router is in a different network/subnet. Home routers also typically include a DHCP server, NAT (Network Address Translation) for sharing one public IP among many private IP devices, and a built-in switch.

7. Gateway

OSI Layer: All layers (Layer 1–7). Connects networks using entirely different protocols (e.g., a LAN to a mainframe network). Performs full protocol translation/conversion. It is the most complex and most capable of all networking devices. In common usage, the term "default gateway" refers to the router's IP address that acts as the exit point from a local network.

8. Modem (Modulator-Demodulator)

OSI Layer: Physical (Layer 1). Converts digital data from a computer into an analogue signal for transmission over telephone/cable lines (modulation), and converts incoming analogue signals back to digital (demodulation). Used to connect to the Internet via ISP infrastructure (DSL, cable, fibre).

Apparatus / Requirements

  • Physical networking devices for observation: NIC, Hub (or visual model), Switch (Layer 2), Router, Patch cables
  • Computer System with Cisco Packet Tracer installed
  • Reference diagrams / datasheet of each device

Procedure

  1. Physically observe each networking device available in the lab: NIC, Hub, Switch, Router, Repeater.
  2. Identify and note: number of ports, LED indicators, type of ports (RJ-45, fibre, console), brand and model number.
  3. Open Cisco Packet Tracer. Place the following devices on the workspace: Hub (generic), Switch (2960), Router (1841 or 2811), Server, 3 PCs.
  4. Hub experiment: Connect 3 PCs to a hub. From PC1, ping PC2. Observe in Packet Tracer's simulation mode that the frame is broadcast to all ports (including PC3).
  5. Switch experiment: Connect the same 3 PCs to a switch. Ping PC2 from PC1. In simulation mode, observe that the frame goes only to PC2 after the MAC table is populated.
  6. Router experiment: Create two different sub-networks (e.g., 192.168.1.0/24 and 192.168.2.0/24). Connect each to a router interface. Assign static IPs to PCs. Configure router interfaces with correct IP addresses. Ping across subnets to confirm routing.
  7. Note down MAC addresses, IP addresses, and routing table entries as observed in Packet Tracer.
  8. Draw a diagram showing the OSI layer position of each device.

Result

Various networking devices — NIC, Repeater, Hub, Bridge, Switch, Router, Gateway, and Modem — were studied in detail. Their working principles, OSI layers, and use cases were understood. Practical simulation in Cisco Packet Tracer confirmed that a hub broadcasts to all ports while a switch forwards only to the destination port. Inter-network communication between two different subnets using a router was also verified successfully.
