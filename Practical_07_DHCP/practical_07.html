Practical No. 07 – DHCP Server Configuration

Aim

To configure a DHCP (Dynamic Host Configuration Protocol) server, assign a DHCP scope/pool, and verify that client computers receive IP addresses automatically using the DORA process.

Theory

DHCP (Dynamic Host Configuration Protocol): An application-layer network protocol (standardised in RFC 2131) that automatically assigns IP addresses and network configuration parameters to devices on a network. It operates on UDP, using port 67 (server) and port 68 (client).

Information provided by DHCP to a client:

  • IP Address (from the configured pool)
  • Subnet Mask
  • Default Gateway
  • DNS Server address(es)
  • IP Address Lease Time

The DORA Process (how a client obtains an IP from a DHCP server):

DDISCOVER
Client broadcasts a request looking for any DHCP server on the network
OOFFER
Server responds with an available IP address offer and configuration info
RREQUEST
Client broadcasts acceptance of the offered IP and requests it formally
AACK
Server confirms the assignment by sending a DHCPACK to the client

Key DHCP Terminology:

TermDefinition
Scope / PoolThe range of IP addresses the server can assign
LeaseThe duration for which an IP address is assigned to a client
ReservationBinding a specific IP always to a specific MAC address
Exclusion RangeIPs within the scope excluded from dynamic assignment (for static devices)

Advantages of DHCP:

  • Eliminates manual IP configuration — reduces human error.
  • Prevents IP address conflicts on the network.
  • Centrally managed — easy to update gateway/DNS for all clients at once.
  • Efficient reuse of IP addresses through leasing.

Apparatus / Requirements

  • Cisco Packet Tracer software (for simulation) OR a Windows Server with DHCP server role
  • 1 Server device configured as DHCP server
  • 3–4 PCs as DHCP clients
  • 1 Switch to connect all devices
  • Straight-through UTP cables

Procedure (Using Cisco Packet Tracer)

  1. Build the Network Topology:
    • Open Cisco Packet Tracer. Create a new workspace.
    • Add 1 Server-PT device (will be the DHCP server).
    • Add 1 Switch (2960 or similar).
    • Add 4 PC devices (will be DHCP clients).
    • Connect the Server and all PCs to the Switch using straight-through cables.
  2. Assign a Static IP to the DHCP Server:
    • Click on the Server → Desktop tab → IP Configuration.
    • Select Static. Enter: IP = 192.168.1.1, Subnet = 255.255.255.0, Gateway = 192.168.1.1.
  3. Configure the DHCP Service on the Server:
    • Click on the Server → Services tab → click DHCP in the left panel.
    • Ensure the DHCP service is turned ON.
    • Configure the pool settings:
      • Pool Name: LAN_POOL
      • Default Gateway: 192.168.1.1
      • DNS Server: 8.8.8.8
      • Start IP Address: 192.168.1.10
      • Subnet Mask: 255.255.255.0
      • Maximum Users: 50
    • Click Add to save the pool, then click Save.
  4. Configure Clients to Use DHCP:
    • Click on PC1 → Desktop → IP Configuration.
    • Select DHCP radio button. Wait a moment.
    • The PC should automatically receive an IP address (e.g., 192.168.1.10) from the pool.
    • Note down the assigned IP, subnet mask, and default gateway.
    • Repeat for PC2, PC3, and PC4. Each should receive a different IP.
  5. Verify Connectivity:
    • On each PC: Desktop → Command Prompt → type ipconfig to verify assigned IP.
    • Ping another PC to verify communication: ping 192.168.1.11
    • All pings should succeed with 0% packet loss.
  6. Document the DHCP pool settings, assigned IPs for each PC, and successful ping results.

Result

A DHCP server was successfully configured in Cisco Packet Tracer with a scope of 192.168.1.10 – 192.168.1.59. All four client PCs received unique IP addresses, subnet masks (255.255.255.0), default gateway (192.168.1.1), and DNS server (8.8.8.8) automatically through the DHCP DORA process. Connectivity between all clients was verified successfully using the ping command.
