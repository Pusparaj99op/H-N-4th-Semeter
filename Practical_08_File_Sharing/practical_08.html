Practical No. 08 – File and Printer Sharing in Windows

Aim

To configure Windows file sharing and printer sharing over a local area network and to access shared resources (files and printer) from a remote computer on the same network.

Theory

File and Printer Sharing: A built-in Windows feature (available in all modern versions) that allows users to share folders, files, drives, and printers over a network so that other authorised users on the same workgroup or domain can access them.

Protocols Used:

  • SMB (Server Message Block): The primary Windows protocol used for file and printer sharing. Also known as CIFS (Common Internet File System). Current versions: SMB 2.0 (Windows Vista+), SMB 3.0 (Windows 8/Server 2012+).
  • NetBIOS: Name resolution service used to discover computers on the local network by their computer name.

Workgroup: A simple peer-to-peer grouping of computers that share the same workgroup name. No centralised server or directory is needed. Each PC maintains its own user accounts and security. Default workgroup in Windows is WORKGROUP.

Share Permissions:

PermissionWhat the User Can Do
ReadView file names, read files/folders, run programs
Change (Read/Write)All Read permissions + create, modify, and delete files
Full ControlAll Change permissions + change permissions for others

UNC (Universal Naming Convention) Path: Standard format to access shared resources over a network: \\ComputerName\ShareName or \\IP_Address\ShareName. Example: \\192.168.1.1\SharedFiles

NTFS vs Share Permissions: NTFS permissions apply to both local and network access; Share permissions apply only to network access. When both are set, the most restrictive permission of the two applies.

Apparatus / Requirements

  • 2 or more Computer Systems running Windows 10 or later
  • LAN network connection (both PCs connected to same switch/router)
  • Both PCs in the same IP subnet and workgroup
  • A test folder with sample files for sharing
  • A printer (physical or virtual PDF printer — optional for Part B)

Procedure

Part A – File Sharing:

  1. Enable Network Discovery and File Sharing on BOTH PCs:
    • Open Control Panel → Network and Sharing Center → Change advanced sharing settings.
    • Under the Private profile: turn on "Network discovery" and "File and printer sharing".
    • Click Save changes.
  2. Verify Both PCs Are in the Same Workgroup:
    • Right-click This PC → Properties → Change settings → Change.
    • Ensure Workgroup = WORKGROUP on both PCs. Restart if changed.
  3. Create a Test Folder on PC1:
    • Open File Explorer and create a folder at C:\Lab_Share.
    • Place 2–3 test files (text files or images) inside it.
  4. Share the Folder from PC1:
    • Right-click C:\Lab_Share → Properties → Sharing tab.
    • Click Share… → In the dropdown, select Everyone → click Add.
    • Set the permission level to Read/Write.
    • Click Share → Note the network path displayed (e.g., \\PRANAY-PC\Lab_Share).
    • Click Done.
  5. Access the Shared Folder from PC2:
    • On PC2, open File Explorer.
    • In the address bar, type: \\[PC1's IP Address] (e.g., \\192.168.1.1) and press Enter.
    • The Lab_Share folder should appear. Double-click to open it.
    • Copy a file from PC1's share to PC2's Desktop to test write/copy access.
    • Create a new text file in the share to test write access.
  6. Verify the file created on PC2 is visible on PC1 in the same shared folder.

Part B – Printer Sharing (Optional):

  1. On PC1 (machine with the printer): Go to Control Panel → Devices and Printers.
  2. Right-click the printer → Printer Properties → Sharing tab.
  3. Check "Share this printer". Give it a share name (e.g., SharedPrinter). Click OK.
  4. On PC2: Go to Control Panel → Devices and Printers → Add a printer.
  5. Select "Add a network, wireless or Bluetooth printer".
  6. PC2 should discover the shared printer. Select and install it.
  7. Print a test page from PC2 to confirm it prints on the shared printer on PC1.

Result

File sharing was successfully configured between two Windows computers connected on the same LAN. The folder C:\Lab_Share on PC1 was shared with Read/Write permissions for Everyone. PC2 successfully accessed the share using the UNC path, read existing files, and created new files in the shared folder. Bidirectional file transfer over the SMB protocol was confirmed. Printer sharing was also demonstrated and a test page was successfully printed from the remote PC.
