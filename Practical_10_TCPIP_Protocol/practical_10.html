Practical No. 10 – Study of TCP/IP Protocol Suite

Aim

To study the TCP/IP protocol suite, understand the four-layer TCP/IP model, compare it with the OSI model, analyse TCP and UDP characteristics, and observe the TCP 3-way handshake using Wireshark.

Theory

TCP/IP Protocol Suite: The foundational set of communication protocols used for the Internet and most computer networks. It was developed by DARPA in the 1970s and consists of two core protocols: TCP (Transmission Control Protocol) and IP (Internet Protocol), along with many supporting protocols.

TCP/IP Model (4 Layers) vs OSI Model (7 Layers):

IP – Internet Protocol: Responsible for logical addressing (IP addresses) and routing of packets from source to destination across multiple networks. IP is connectionless and unreliable — it makes no guarantee of delivery, ordering, or error correction. IPv4 uses 32-bit addresses; IPv6 uses 128-bit addresses.

TCP – Transmission Control Protocol:

  • Connection-oriented: A connection must be established before data is sent.
  • Reliable: Guarantees delivery using acknowledgements, retransmission, and sequencing.
  • Flow control: Uses a sliding window mechanism to match send rate with receiver capacity.
  • Congestion control: Adjusts transmission rate to avoid network congestion.
  • Used by: HTTP, HTTPS, FTP, SMTP, SSH.

UDP – User Datagram Protocol:

  • Connectionless: No handshake; data is sent without establishing a connection.
  • Unreliable: No guarantee of delivery, ordering, or duplicate protection.
  • Faster and lower overhead than TCP.
  • Used by: DNS, DHCP, video streaming, VoIP, online gaming.

TCP 3-Way Handshake – How TCP establishes a connection:

  1. SYN: Client sends a TCP segment with SYN (synchronise) flag set, along with a random Initial Sequence Number (ISN).
  2. SYN-ACK: Server receives the SYN, replies with SYN+ACK. Acknowledges client's ISN and sends its own ISN.
  3. ACK: Client sends a final ACK to acknowledge the server's ISN. Connection is now established.

Connection is terminated using a 4-way handshake: FIN → ACK → FIN → ACK.

Apparatus / Requirements

  • Computer System with Windows OS
  • Wireshark (packet analyser – free download)
  • Command Prompt
  • Active internet/LAN connection

Procedure

  1. Open Wireshark. Start packet capture on the active Ethernet/Wi-Fi interface.
  2. Open a browser and visit any HTTP website (e.g., http://example.com). Return to Wireshark and stop capture.
  3. In the filter box, type tcp and press Enter to show only TCP packets.
  4. Observe the TCP 3-Way Handshake:
    • Find the first packet from your machine — it should be a [SYN] packet.
    • The next packet from the server should be [SYN, ACK].
    • The third packet from your machine should be [ACK]. This completes the handshake.
  5. Click on the SYN packet. Expand the Transmission Control Protocol section in the packet details pane. Record:
    • Source Port and Destination Port
    • Sequence Number and Acknowledgement Number
    • Flags set (SYN, ACK)
    • Window Size
  6. In the filter box, type udp to view UDP traffic. Click a DNS packet (port 53) and note the smaller header size and lack of sequence/acknowledgement numbers compared to TCP.
  7. In Command Prompt, run netstat -n to observe active TCP connections with their port numbers and states (ESTABLISHED, TIME_WAIT).
  8. Document all findings with packet details and screenshots.

Result

The TCP/IP protocol suite was studied in detail. The 4-layer TCP/IP model and its mapping to the OSI model were understood. Key characteristics of TCP (reliable, connection-oriented, 3-way handshake) and UDP (unreliable, connectionless, faster) were compared. Using Wireshark, the TCP 3-way handshake (SYN → SYN-ACK → ACK) was observed in a live capture, and well-known port numbers for HTTP, DNS, FTP, and DHCP were identified.
