Practical No. 12 – FTP Server Configuration

Aim

To configure an FTP (File Transfer Protocol) server using Cisco Packet Tracer, create FTP user accounts with permissions, and demonstrate file upload (put) and download (get) operations from an FTP client.

Theory

FTP – File Transfer Protocol: An Application Layer (Layer 7) protocol used to transfer files between a client and server over a TCP/IP network. Defined in RFC 959. FTP uses two separate TCP connections:

  • Port 21 – Control Connection: Used to send commands and responses (authentication, navigation, file listing, etc.). Remains open for the entire session.
  • Port 20 – Data Connection: Used to transfer the actual file data. Opened and closed for each individual transfer.

FTP Modes of Data Transfer:

Active Mode Server connects back to the client on a client-specified port (PORT command) to open the data channel. May be blocked by client-side firewalls.
Passive Mode (PASV) Server opens a random port for data; client connects to it. Preferred mode today — more firewall-friendly as only client initiates connections.

FTP vs SFTP vs FTPS:

Protocol Encryption Port Security
FTPNone (plaintext)21 / 20Credentials sent in clear text — NOT secure
FTPSSSL/TLS21 (implicit: 990)FTP with TLS encryption layer — secure
SFTPSSH22Completely different protocol over SSH — most secure

Common FTP Commands:

FTP CommandDescription
ftp [IP]Connect to FTP server at the given IP address
open [IP]Open a connection to an FTP server
ls / dirList files and directories on the server
get [filename]Download a file from the server to the local machine
put [filename]Upload a file from the local machine to the server
cd [dir]Change directory on the server
mkdir [dir]Create a new directory on the server
delete [file]Delete a file from the server
bye / quitDisconnect from the FTP server and exit

FTP User Permissions in Packet Tracer:

  • Read: View and download files only.
  • Write: Upload files; create directories.
  • Delete: Remove files from the server.
  • Rename: Rename files on the server.
  • List: List directory contents.

Apparatus / Requirements

  • Cisco Packet Tracer software
  • 1 Server-PT (FTP Server), 1 Switch, 2 PC devices
  • Straight-through UTP cables
  • A test file pre-loaded on the server for download demonstration

Procedure

  1. Build the Network Topology:
    • Add 1 Server-PT, 1 Switch (2960), and 2 PCs. Connect all to the switch using straight-through cables.
  2. Assign Static IP to the FTP Server:
    • Click Server → Desktop → IP Configuration → Static.
    • IP: 192.168.1.1, Subnet Mask: 255.255.255.0.
  3. Configure the FTP Service:
    • Click Server → Services → FTP. Ensure FTP is turned ON.
    • Create a new FTP user:
      • Username: pranay
      • Password: lab123
      • Check all permissions: Read, Write, Delete, Rename, List.
      • Click Add.
    • Note the default file sampleFile.txt that exists on the server. If not visible, verify server files in the FTP files area.
  4. Configure PC1 (FTP Client):
    • Desktop → IP Configuration → Static. IP: 192.168.1.10, Subnet: 255.255.255.0.
  5. Configure PC2: IP: 192.168.1.20, Subnet: 255.255.255.0.
  6. Connect to FTP Server from PC1:
    • Desktop → Command Prompt. Type: ftp 192.168.1.1
    • Enter username: pranay
    • Enter password: lab123
    • You should see: 230 – Logged in.
  7. FTP Operations:
    • List files on server: dir — note the available files.
    • Download a file: get sampleFile.txt — file is transferred to PC1.
    • Upload a file: put testReport.txt — (a file that exists on PC1's flash memory). Note: In Packet Tracer, the file must exist on the virtual PC.
    • Rename a file: rename sampleFile.txt backupFile.txt
    • Delete a file: delete backupFile.txt
    • Verify operations: dir — confirm changes.
    • Disconnect: bye
  8. Repeat the connection from PC2 to verify multi-client access to the same FTP server.
  9. Document all FTP commands used and the server's response codes:
    CodeMeaning
    220Service ready (welcome message)
    230User logged in successfully
    226Transfer complete
    227Entering passive mode
    331Username OK, password required
    530Not logged in / authentication failed
    550File unavailable / not found
