Practical No. 11 – DNS Configuration and Study

Aim

To study the Domain Name System (DNS), understand the DNS resolution process, configure a DNS server in Cisco Packet Tracer, and verify name-to-IP resolution from client machines using nslookup and ping.

Theory

DNS – Domain Name System: A hierarchical, distributed database that translates human-readable domain names (e.g., www.google.com) into machine-readable IP addresses (e.g., 142.250.195.100) and vice versa. It operates on UDP port 53 (and TCP port 53 for larger responses). DNS is essential for the usability of the internet — without it, users would need to remember numeric IP addresses.

DNS Record Types:

Record Type Full Name Purpose / Example
AAddress RecordMaps a hostname → IPv4 address. e.g., www.abc.com → 192.168.1.100
AAAAIPv6 Address RecordMaps a hostname → IPv6 address
CNAMECanonical NameAlias record; maps one hostname to another. e.g., mail.abc.com → abc.com
MXMail ExchangeSpecifies the mail server for a domain
NSName ServerSpecifies the authoritative DNS server for a domain
PTRPointer RecordReverse DNS lookup: maps IP → hostname
SOAStart of AuthorityAdministrative info about a DNS zone

DNS Hierarchy and Resolution Process:

When a user types www.google.com in a browser, DNS resolution occurs in the following order:

1. Browser Cache
Checks local browser DNS cache
→
2. OS Cache
Checks OS/hosts file DNS cache
→
3. Recursive Resolver
Typically your ISP's or configured DNS (e.g. 8.8.8.8)
→
4. Root Server
Refers to TLD (.com) server
→
5. TLD Server
Refers to google.com authoritative server
→
6. Authoritative Server
Returns the IP address for www.google.com

hosts File: A local text file (C:\Windows\System32\drivers\etc\hosts on Windows) that can be manually edited to map hostnames to IP addresses, bypassing DNS. It is checked before DNS queries. Priority: hosts file > DNS cache > DNS server.

Apparatus / Requirements

  • Cisco Packet Tracer software
  • 1 Server-PT (DNS Server), 1 Switch, 3 PC devices
  • Straight-through cables
  • Computer with internet access (for live nslookup demonstration)
  • Command Prompt (Windows)

Procedure

Part A — Configure DNS in Cisco Packet Tracer:

  1. Open Cisco Packet Tracer and create a new topology:
    • Add 1 Server-PT, 1 Switch (2960), and 3 PCs. Connect all to the switch.
  2. Configure the DNS Server's static IP:
    • Click Server → Desktop → IP Configuration → Static.
    • IP: 192.168.1.1, Subnet: 255.255.255.0, Gateway: 192.168.1.1.
  3. Configure the DNS Service on the Server:
    • Click Server → Services → DNS. Turn the DNS service ON.
    • Add DNS records (A Records):
      • Name: www.pranay.com → Address: 192.168.1.10 → Click Add
      • Name: www.college.com → Address: 192.168.1.20 → Click Add
    • Click Save.
  4. Configure PC1 (client):
    • IP: 192.168.1.10, Subnet: 255.255.255.0, Gateway: 192.168.1.1, DNS: 192.168.1.1.
  5. Configure PC2: IP: 192.168.1.20, same subnet/gateway/DNS.
  6. Configure PC3: IP: 192.168.1.30, same subnet/gateway/DNS.
  7. Test DNS Resolution from PC3:
    • Desktop → Command Prompt → type: ping www.pranay.com
    • DNS should resolve to 192.168.1.10 and ping should succeed.
    • Type: nslookup www.college.com → Should return 192.168.1.20.

Part B — Live DNS Commands (Windows CMD):

  1. Open CMD. Type nslookup www.google.com — note the resolved IP address and which DNS server answered.
  2. Type nslookup 8.8.8.8 — observe the reverse DNS (PTR) lookup result.
  3. Type ipconfig /displaydns — view the local DNS resolver cache entries.
  4. Type ipconfig /flushdns — clear the DNS cache. Confirm with ipconfig /displaydns again.
  5. Open Notepad as Administrator. Open the hosts file at C:\Windows\System32\drivers\etc\hosts. Add a test entry: 192.168.1.100 testsite.local. Save.
  6. In CMD, run ping testsite.local — it should resolve to 192.168.1.100 from the hosts file, bypassing DNS.
  7. Remove the test entry from the hosts file after verification.

Result

The DNS protocol was studied in detail — its structure, record types, and step-by-step resolution process were understood. A DNS server was successfully configured in Cisco Packet Tracer with two A records. Client PCs resolved hostnames (www.pranay.com, www.college.com) to their correct IP addresses using the configured DNS server. Live DNS commands (nslookup, ipconfig /displaydns, ipconfig /flushdns) were also demonstrated on a Windows machine.
